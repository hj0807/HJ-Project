# Builder Pattern



## 1. Builder Pattern 정의

- 객체의 `생성과 표현을 분리`시킨다.

- 하나의 객체를 생성할 때, 객체를 이루고 있는 구성 요소들을 분리하여 세부 구성요소 클래스들을 별도로 만들고, 그 구성 요소 클래스들을 조합하여 하나의 객체를 만든다.

  ​

## 2. Bridge Pattern 의 사용 이유

- 인터페이스와 구현부가 서로 독립적인 구조를 가지면서 확장하기를 원할때 사용
